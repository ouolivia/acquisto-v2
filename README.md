# 采易单

一款可发布到公网、安装到手机并离线使用的采购应用，支持采购录入、颜色与门店批量分配、明细修改，以及 Excel / PDF 导出。

## 已实现

- 供应商名称与采购日期（默认当天）
- 型号、进价、卖价录入；卖价只输入整数时自动补 `.99`
- 按“件 / 包”采购；选择“包”时记录每包件数
- 预设颜色：`-1`、`-2`、`-13`、`nero`、`bianco`、`黑`、`白`
- 可随时增加临时颜色
- 预设 1–20 号门店，可多选或全选
- 一次按“颜色 × 门店”批量生成采购明细
- 同一型号可只改颜色、数量和门店后继续分配
- 明细按型号汇总，可修改或删除单条分配
- 显示型号数、总件数和采购金额
- 导出 Excel 可打开的 `.xls` 表格
- 导出带中文内容的 `.pdf` 表格
- 数据保存在当前手机浏览器中，支持离线使用

## 本机预览

在本目录启动任意静态文件服务后打开 `index.html`。例如：

```bash
python3 -m http.server 4173
```

然后访问 `http://localhost:4173`。

## 发布到公网（GitHub Pages）

项目已包含 `.github/workflows/deploy-pages.yml`，推送到 GitHub 后会自动发布：

1. 在 GitHub 新建一个仓库，例如 `acquisto`。
2. 将本目录全部文件推送到仓库的 `main` 分支。
3. 打开仓库 `Settings → Pages`。
4. 在 `Build and deployment → Source` 选择 `GitHub Actions`。
5. 等待 Actions 中的 `Deploy procurement app to GitHub Pages` 完成。

公网地址通常为：`https://你的GitHub用户名.github.io/仓库名/`。

## 安装到手机桌面

将本目录部署到一个 HTTPS 静态网站后：

- iPhone Safari：分享 → 添加到主屏幕
- Android Chrome：菜单 → 安装应用 / 添加到主屏幕

安装后可全屏使用；首次打开完成缓存后，没有网络也能继续录入和查看数据。

## 数据说明

采购数据只保存在当前手机、当前浏览器、当前网址的本地存储中。更换网址、浏览器或手机不会自动同步数据；清除浏览器网站数据也会删除采购记录。建议每批采购完成后及时导出 Excel 或 PDF 备份。
